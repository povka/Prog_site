# Prog with the Bois

A static **Yu-Gi-Oh! progression tracker** for weekly results, decklists, and player binders.

This project is built as a simple front-end in `dist/` with generated binder data under `dist/data/generated/`. It is meant to be easy to browse, easy to update, and easy to host.

---

## What this project does

**Prog with the Bois** is a lightweight hub for a Yu-Gi-Oh! progression group.

It brings together:

- **Weekly results**
- **Decklist browsing**
- **Player binder viewing**
- **Card metadata and images**
- **Automated binder refreshes**

The goal is to keep progression info in one place instead of spreading it across screenshots, chats, and random folders.

---

## Features

- **Weekly Results Hub** for standings, winners, format notes, and rewatch links
- **Deck Browser** for browsing decklists by week / format
- **Binder Viewer** with card filtering for things like:
  - card type
  - attribute
  - race / monster type
  - subtype
  - ATK / DEF / Level / Rank / Link Rating / Scale
- **Generated binder JSON** so the front-end can stay static and fast
- **Automated updates** through GitHub Actions
- **Discord notifications** when binder changes are detected

---

## How it works

The site itself is static, but the binder data is generated by a Python script.

### Data flow

1. The script logs into configured YGOProg accounts.
2. It pulls binder data from the YGOProg binder API.
3. It fetches card metadata from **YGOProDeck**.
4. It builds a local card index cache.
5. It writes normalized output into `dist/data/generated/`.
6. GitHub Actions commits updated generated files when changes are found.
7. A Discord bot posts binder change summaries.

This keeps the front-end simple while still allowing the data to refresh automatically.
